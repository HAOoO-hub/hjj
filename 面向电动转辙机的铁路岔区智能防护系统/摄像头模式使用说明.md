---

## 1. 摄像头模式需要修改哪些参数

### 必改 1：切换成摄像头模式
把：
```python
"input_mode": "video",
```

改成：
```python
"input_mode": "camera",
```

---

### 必改 2：摄像头 RTSP 地址
在 `CONFIG["camera"]` 里修改：

```python
"rtsp_url": "rtsp://127.0.0.1:8554/z4_train",
```

把它改成你自己的摄像头 RTSP 地址。
例如：

```python
"rtsp_url": "rtsp://admin:password@192.168.1.64:554/h264/ch2/sub/av_stream",
```

---

### 必改 3：设备编号
在 `CONFIG["camera"]` 里修改：

```python
"device_id": 1,
```

这个值会在传接口时作为 `deviceId` 发出去。  
所以要和你后端系统里的摄像头编号保持一致。

---

### 必改 4：接口地址
如果你要把结果传给接口，需要检查这几项：

```python
"api_push_enabled": True,
"external_api": {
    "base_url": "http://test.agilefast.cn:8087/SmartAI",
    "endpoint": "/SmartAI/ai/objectdetection/pushRailStatus",
    "device_id_param": "deviceId",
    "status_param": "railStatus",
}
```

你通常只需要确认：

- `api_push_enabled` 是否开启，开启状态用于传外部api接口
- `base_url` 是否是你的服务地址
- `endpoint` 是否正确

## 2. 这份代码默认用什么模型

这份代码当前默认配置是：

```python
"backend": "pytorch",
"model_path": r"model_data/l_train_person.pth",
"classes_path": r"model_data/cls_classes_train_person.txt",
"phi": "l",
```

也就是说：

- 推理后端：`pytorch`
- 模型文件：`model_data/l_train_person.pth`
- 类别文件：`model_data/cls_classes_train_person.txt`
- 模型规格：`phi = "l"`

---

## 3. 传接口的 state 是什么意思

接口上传的是 `railStatus`。  
代码里的状态定义是：

```python
STATE_CLEAR = 0
STATE_WARNING = 1
STATE_TRAIN = 2
STATE_PERSON = 3
```

所以传给接口时：

- `0` = `CLEAR`：正常，无人、无车进入目标轨道
- `1` = `WARNING`：疑似有物品靠近目标铁轨
- `2` = `TRAIN`：火车进入/占用目标轨道
- `3` = `PERSON`：人上轨 / 人进入目标轨道，最高优先级

---

## 5. 最少要检查的参数清单

如果你只想最快跑起来，摄像头模式下至少检查这几项：

```python
"input_mode": "camera"
"camera": {
    "device_id": 1,
    "rtsp_url": "你的RTSP地址",
    "output_video_path": r"output/camera_result.mp4",
}
"backend": "pytorch"
"model_path": r"model_data/m_train_person.pth"
"classes_path": r"model_data/cls_classes_train_person.txt"
"phi": "m"
"api_push_enabled": True
```

如果接口也要通，再确认：

```python
"external_api": {
    "base_url": "你的接口地址",
    "endpoint": "你的接口路径",
    "device_id_param": "deviceId",
    "status_param": "railStatus",
}
```

---

## 6. 一句话总结

用摄像头模式时，最关键就是改这几个地方：

- `input_mode = "camera"`
- `camera.rtsp_url`
- `camera.device_id`
- `external_api.base_url` / `endpoint`
- 模型路径 `model_path`

接口上传的 `railStatus` 含义是：

- `0` 表示没有物体出现在目标铁轨
- `1` 疑似有物品靠近目标铁轨
- `2` 表示有火车在目标铁轨出现
- `3` 表示有人在目标铁轨出现
